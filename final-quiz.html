Q1
Which condition MUST be true before using binary search?
✅ Correct. Binary search only works on sorted data. It relies on eliminating half the list each step based on order.
❌ Binary search requires sorted data. Without sorting, the "eliminate half" logic breaks down.
Q2
What is the time complexity of binary search in the worst case?
✅ Correct. Binary search is O(log n) — halving the search space each step means 1000 elements need at most ~10 comparisons.
❌ Binary search is O(log n) — much faster than O(n) linear search for large datasets.
Q3
In bubble sort, what does the swapped flag optimisation do?
✅ Correct. If no swaps occur in a full pass, the list is already sorted. The flag stops unnecessary further passes.
❌ The swapped flag is an early-exit optimisation. No swaps = sorted. It doesn't change the worst-case complexity.
Q4
What are the TWO essential parts of every recursive function?
✅ Correct. The base case stops recursion; the recursive case reduces the problem size each call.
❌ Every recursive function needs a base case (stop condition) and a recursive case (self-call with smaller input).
Q5
A Stack uses which access principle?
✅ Correct. A Stack is LIFO — the last item pushed is the first item popped. Think of a stack of trays.
❌ Stacks are LIFO. Queues are FIFO. Remember: Stack = pile of plates; Queue = queue at a bus stop.
Q6
In OOP, what is encapsulation?
✅ Correct. Encapsulation protects data by making attributes private and providing getters/setters for controlled access.
❌ Encapsulation = hiding data. Inheritance = parent/child. Polymorphism = same method, different behaviour.
Q7
What is the best case time complexity for insertion sort?
✅ Correct. When already sorted, insertion sort only needs one comparison per element → O(n). This makes it great for nearly-sorted data.
❌ Best case for insertion sort is O(n) — when the list is already sorted, no shifting is needed.
Q8
Which file mode in Python will ADD data to the end of an existing file without deleting its contents?
✅ Correct. "a" = append mode. It opens the file and writes at the end. "w" would overwrite everything.
❌ "a" is append mode. "r" = read only. "w" = write (overwrites). Use "a" when you want to keep existing data.
