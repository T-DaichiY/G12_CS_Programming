Searching — Binary Search
①
🔍 What is Binary Search?
Binary search works on a sorted list only. It finds the middle element, then eliminates half of the remaining list each time. This makes it dramatically faster than linear search for large datasets.
Condition Required
Sorted list
Binary search ONLY works if the data is sorted first.
Time Complexity
O(log n)
Each step halves the search space. 1000 items → max 10 comparisons.
⚠️ Common exam mistake
Students often forget that binary search requires a sorted array. If data is unsorted, you must sort it first — or use linear search. Examiners specifically test this condition.
②
📋 Pseudocode Logic
Pseudocode — Binary Search
FUNCTION BinarySearch(arr, target) low ← 0 high ← LENGTH(arr) - 1 WHILE low <= high mid ← (low + high) DIV 2 IF arr[mid] = target THEN RETURN mid ELSE IF arr[mid] < target THEN low ← mid + 1 // search right half ELSE high ← mid - 1 // search left half ENDIF ENDWHILE RETURN -1 ENDFUNCTION
③
🐍 Python Implementation
Python — binary_search.py
def binary_search(arr, target): """arr must be sorted. Returns index or -1.""" low = 0 high = len(arr) - 1 while low <= high: mid = (low + high) // 2 if arr[mid] == target: return mid elif arr[mid] < target: low = mid + 1 else: high = mid - 1 return -1 # Must be sorted! data = [3, 8, 15, 27, 42, 56] print(binary_search(data, 27)) # Output: 3 print(binary_search(data, 10)) # Output: -1

🆓 L3 — Free Style

Write binary search from memory. Then add a second function count_steps(arr, target) that returns how many comparisons were needed. Test on an array of 100 sorted numbers.

💡 Hint: Use a counter variable inside the while loop. Increase it each iteration.
Answer
def binary_search(arr, target): low, high = 0, len(arr) - 1 while low <= high: mid = (low + high) // 2 if arr[mid] == target: return mid elif arr[mid] < target: low = mid + 1 else: high = mid - 1 return -1 def count_steps(arr, target): low, high = 0, len(arr) - 1 steps = 0 while low <= high: mid = (low + high) // 2 steps += 1 if arr[mid] == target: return steps elif arr[mid] < target: low = mid + 1 else: high = mid - 1 return steps nums = list(range(0, 200, 2)) # 100 sorted even numbers print(f"Index: {binary_search(nums, 150)}") print(f"Steps: {count_steps(nums, 150)}") # at most 7 steps
🎯 L4 — Scenario
Situation
A hospital stores patient IDs as sorted integers. The system receives thousands of lookup requests per second. Implement a binary search for patient ID lookup and compare its performance against linear search by counting comparisons on a list of 1000 IDs.
  • Create a sorted list of 1000 patient IDs (use list(range(1000, 2000)))
  • Implement both linear and binary search, each counting comparisons
  • Search for ID 1750 and print comparison counts for both
  • Add a comment: explain why binary search is preferred here
  • What happens if you search an unsorted list with binary search? Test it.
Answer
import random def linear_search(arr, target): for i, val in enumerate(arr): if val == target: return i, i + 1 return -1, len(arr) def binary_search(arr, target): low, high = 0, len(arr) - 1 comparisons = 0 while low <= high: mid = (low + high) // 2 comparisons += 1 if arr[mid] == target: return mid, comparisons elif arr[mid] < target: low = mid + 1 else: high = mid - 1 return -1, comparisons ids = list(range(1000, 2000)) l_idx, l_count = linear_search(ids, 1750) b_idx, b_count = binary_search(ids, 1750) print(f"Linear: index {l_idx}, comparisons: {l_count}") print(f"Binary: index {b_idx}, comparisons: {b_count}") # Binary: ~10 steps vs Linear: ~751 steps unsorted = list(range(1000, 2000)) random.shuffle(unsorted) b2_idx, b2_count = binary_search(unsorted, 1750) print(f"Unsorted binary: found={b2_idx!=-1}, steps={b2_count}") # UNRELIABLE — binary search requires sorted data!
