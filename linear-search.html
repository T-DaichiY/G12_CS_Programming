Searching — Linear Search
①
🔍 What is Linear Search?
Linear search checks each element one by one from the start until it finds the target or reaches the end. It works on any list — sorted or unsorted. It is the simplest search algorithm, but also the slowest for large datasets.
Best Case
O(1)
Target is the very first element. Only 1 comparison needed.
Worst / Average Case
O(n)
Target is last or not found. Must check every element.
💡 Simple analogy
Imagine looking for a name on a class register — you start from the top and read each name until you find it. That's linear search. No sorting needed, but it can take a long time.
②
📋 Pseudocode Logic
The algorithm loops through each position. If the element matches, return the index. If the loop ends without a match, return -1 (not found).
Pseudocode — Linear Search
FUNCTION LinearSearch(arr, target) FOR i ← 0 TO LENGTH(arr) - 1 IF arr[i] = target THEN RETURN i // found: return index ENDIF NEXT i RETURN -1 // not found ENDFUNCTION
③
🐍 Python Implementation
Python — linear_search.py
def linear_search(arr, target): """Search for target in arr. Return index or -1.""" for i in range(len(arr)): if arr[i] == target: return i # found return -1 # not found # --- Test --- data = [15, 3, 42, 8, 27] print(linear_search(data, 42)) # Output: 2 print(linear_search(data, 99)) # Output: -1

Write a linear search that searches a list of strings (not numbers). It should be case-insensitive (so searching "apple" finds "Apple"). Return the index or -1.

💡 Hint: Use .lower() to convert both strings before comparing.
Python — Write your code here
# Goal: case-insensitive linear search on a list of strings # Your function here: # Test: fruits = ["Apple", "Banana", "Cherry"] # search for "banana" should return 1 # search for "grape" should return -1
Answer
def linear_search_ci(lst, target): target_lower = target.lower() for i, item in enumerate(lst): if item.lower() == target_lower: return i return -1 fruits = ["Apple", "Banana", "Cherry", "Date", "Elderberry"] print(linear_search_ci(fruits, "apple")) # 0 print(linear_search_ci(fruits, "CHERRY")) # 2 print(linear_search_ci(fruits, "banana")) # 1 print(linear_search_ci(fruits, "grape")) # -1
🎯 L4 — Scenario: Real-world application
Situation
A school library stores book ISBNs in an unsorted list. When a student scans a barcode, the system must check whether the book exists. Write a complete program that: loads a list of ISBNs, performs a linear search, and prints a friendly message.
  • Create a list of at least 6 ISBN strings (e.g. "978-0-13-110362-7")
  • Write a find_book(library, isbn) function using linear search
  • Return the index and print "Book found at position X" or "Book not in library"
  • Test with both a valid and an invalid ISBN
  • Add a comment explaining why binary search could NOT be used here
Answer
def find_book(library, isbn): for i, book_isbn in enumerate(library): if book_isbn == isbn: return i return -1 library = [ "978-0-13-110362-7", "978-0-20-161622-4", "978-0-59-651798-1", "978-0-13-468599-1", "978-0-13-235088-4", "978-0-44-01077-5" ] result = find_book(library, "978-0-59-651798-1") if result != -1: print(f"Book found at position {result}") else: print("Book not in library") result2 = find_book(library, "999-9-99-999999-9") if result2 != -1: print(f"Book found at position {result2}") else: print("Book not in library") # Binary search CANNOT be used: the list is unsorted. # Binary search requires sorted data to work correctly.
📝 Key Terms
Linear Search
sequential search
Check each element one by one from the start
O(n)
time complexity
Performance grows linearly with data size
index
position number
The position of the element in the array (starts at 0)
-1
not found signal
Convention: return -1 when target is not in the list
